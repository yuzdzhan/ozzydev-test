Modern CSS Tricks I Reach For Daily

O OzzyDev Team May 28, 2026 5 min read

CSS quietly got incredible. A lot of the JavaScript and preprocessor hacks we relied on for years are now one-liners in plain CSS — shipping in every evergreen browser. Here are the features that earned a permanent spot in our toolbox.

Container queries

Media queries respond to the viewport. Container queries respond to the component's own width — so a card can lay itself out based on the space it's actually given, not the whole screen. This is the real win for reusable components. 

.card-wrap { container-type: inline-size; }

/* When the CARD is wider than 400px, go horizontal */
@container (min-width: 400px) {
  .card { display: grid; grid-template-columns: 120px 1fr; }
}

The :has() parent selector

For two decades CSS could only style children based on parents. :has() finally lets a parent react to its children — the long-awaited "parent selector." 

/* Style a form field that contains an invalid input */
.field:has(input:invalid) { border-color: crimson; }

/* A card with no image gets extra padding */
.card:not(:has(img)) { padding-block: 2rem; }
Underrated power move: :has() can replace a surprising amount of JavaScript that exists only to toggle a class on a parent element.

color-mix() for theming

Need a tint of your brand color for a hover state or a translucent background? Mix it in CSS — no extra variables, no preprocessor functions. 

:root { --brand: #58a6ff; }

.badge {
  background: color-mix(in srgb, var(--brand) 15%, transparent);
  color: var(--brand);
}
.btn:hover {
  background: color-mix(in srgb, var(--brand) 85%, black);
}

Fluid type with clamp()

One line scales a font smoothly between a minimum and maximum size — no breakpoints required. 

h1 {
  /* min 2rem, scales with viewport, never exceeds 3.6rem */
  font-size: clamp(2rem, 5vw, 3.6rem);
}

Logical properties

Use margin-inline and padding-block instead of left/right/top/bottom and your layouts adapt automatically to right-to-left languages and vertical writing modes. 

.container {
  margin-inline: auto;     /* left + right */
  padding-block: 2rem;     /* top + bottom */
}

Scroll-driven niceties

Smooth scrolling and snap points need zero JavaScript now. 

html { scroll-behavior: smooth; }

.gallery {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
}
.gallery > * { scroll-snap-align: center; }
The best CSS trick in 2026 is realizing how little CSS you now need to write.

Most of these are already in the stylesheet powering this very blog. Pop open your dev tools and have a look — then go delete some JavaScript.
